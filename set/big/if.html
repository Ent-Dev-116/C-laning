if(ついでにelse)

ifの入力の仕方

ifはif(（　①　）){　②　};のような入力法になります。

①

①には条件式が入ります。条件式とは、if文の{}に入るための条件です。

②

②の中には、条件式が成功したときに実行するプログラムを書きます。

else

elseとは、条件式に失敗したプログラムを実行するものです。
if(){}else{　③　};と記述し、③には実行するプログラムを書きます。

条件式一覧

条件式を一覧にして表を載せました。

入力 実例 成功の条件
> a > b [a]が[b]より大きければ成功
< a < b [a]が[b]より小さければ成功
== a == b [a]と[b]が等しければ成功
!= a != b [a]と[b]が等しくなければ成功
>= a >= b [a]が[b]より大きい、もしくは等しければ成功
<= a <= b [a]が[b]より小さい、もしくは等しければ成功
|| a == b || a == c 「または」という意味の記号。この場合は「[a]が[b]と同じ」または「[a]が[c]と同じ」なら成功。
&& a > b && a < c 「かつ」という意味の記号。この場合は「[a]が[b]より大きい」かつ「[a]が[c]より小さい」なら成功。
! !(a > b) 「ではない」という意味の記号。この場合は「[a]が[b]より大きい」のではないなら成功。

今までの説明を分かりやすくするサンプルコード

#include<stdio.h>

/**********************************************
    if...else 説明用サンプルコード          
    作成者:換気扇  
    推奨ファイル名:if_sanp.c          
**********************************************/

int main(void){
        
        //変数宣言
        int num;

        //値を代入
        printf("num\n>");
        scanf("%d",&num);

        //if文の性質証明
        if(1){
            printf("\nifは比較式が成功するだけでなく、()の中身が「1」であれば通ります。\n");
        }

        //if文の比較式
        if(num>10){
            printf("\n入力した数値は10より大きいです。\n");
        }

        //if文とelse文の使用
        if(num%2==1){
            printf("入力した数値は奇数です。\n");
        } else {
            printf("入力した数値は偶数です。\n");
        }

        //if文の複合比較式
        if(num>10&&num<20){
            printf("\n入力した数値は10より大きいく、20より小さいです。\n");
        }


        return 0;
}

サンプルコード徹底解説

ここでは、サンプルコードで果たして何をしていたのかを表します。

1,#include ... (void){～定型文～

ザ・定型文です。
今のうちは、「始めの合図」だと思ってください。

2,int num;～変数宣言～

変数宣言です。一応変数の説明をするならば、入力した数値を保存する変数です。

3,printf("num ... %d",&num);～数値入力～

if文で比較するための数値です。この値で今後のif文で分岐が変わります。

4,if(1){ ... }～if文の性質～

if文はコメントの通り、比較式だけでなく()の中身が「1」であれば通ります。
この技術はifだけでなく、今後のwhileの方でよく使うので忘れない方が良いです。

5,if(num>10){ ... }～if文の比較式のテスト～

この比較式であれば入力した値が10より大きければ中身に入ります。

6,if(num%2==1){ ... }～if文の比較式のテスト～

if...else文のテスト。もし「入力した数値を2で割った余りが1(奇数)」ならif文の中に入り、でなければelse文の中に入る。

7,if(num>10&&num<20){ ... }～if文の複合した比較式のテスト～

この比較式であれば、「numが10より大きく、かつ、20より小さい」場合にif文の中に入る。

8,retu ... }～定型文～

「1,」の内容が始めの合図だとすると、こっちは「終わりの合図」です。
「1,」と「8,」が何なのかは追々分かります。

オマケ[ifでよくあるエラー三選]

1.比較式と代入

等しいの場合に、==でなく=の記述をしてしまうと発生します。これは何も発生しなくなります。
エラーではなく、バグとされてしまうので、発見も難しいです。

2.比較式の間違い

長文の分岐を書いている時に、どの分岐でどこへ入るのかわかりづらくなる時があります。
それを避けるためにコメント文はちゃんと残しておきましょう。

3.中かっこつけ忘れ

手癖でプログラムを書いている時にあるミスです。
中かっこを閉じ忘れただけでめちゃくちゃ分かりづらいエラーをはいてくるので、避けた方が良いです。
